## 第一章：产品概述与定位

### 1.1 产品愿景与目标用户

#### 1.1.1 产品定位

Claude Code 调度平台是一个面向团队协作的 **AI Agent 任务调度平台**。它将 Claude Code CLI 的单会话能力扩展为多 Agent 并行协作模式，通过看板式界面实现任务的分配、跟踪、干预和管理。

**核心价值主张**：

| 价值维度 | 描述 |
|----------|------|
| **并行效率** | 将原本串行执行的任务分解为多个 Agent 并行处理，显著提升工作效率 |
| **专业分工** | 不同类型的 Agent（Explore、Plan、Verification）各司其职，发挥专业优势 |
| **可视管理** | 看板式界面提供直观的任务状态展示，降低认知负担 |
| **实时干预** | 用户可随时介入 Agent 执行过程，调整方向或处理权限请求 |
| **团队协作** | 支持多用户共享 Agent 资源，实现团队级别的任务协作 |

#### 1.1.2 目标用户画像

**主要用户群体**：

1. **软件开发团队负责人**
   - 典型场景：分配代码审查任务、监控修复进度、管理多个并行开发任务
   - 核心需求：任务分配、进度追踪、结果审核
   - 使用频率：每日高频使用

2. **研究人员与技术探索者**
   - 典型场景：同时探索多个技术方向、并行研究多个代码库、对比不同方案
   - 核心需求：并行执行、结果对比、方向调整
   - 使用频率：项目周期内持续使用

3. **DevOps 工程师**
   - 典型场景：同时管理多个运维任务、监控部署状态、处理紧急告警
   - 核心需求：实时监控、快速干预、任务取消
   - 使用频率：运维事件期间高频使用

4. **独立开发者**
   - 典型场景：管理个人项目的多线程任务、自动化重复性工作
   - 核心需求：任务组织、进度追踪、效率提升
   - 使用频率：项目开发期间日常使用

**用户能力假设**：
- 具备基本的软件开发知识
- 了解 Claude Code 的基本使用方式
- 理解 Git 工作流和任务管理概念

#### 1.1.3 产品边界定义

**核心功能边界**：
- ✅ 多 Agent 创建与管理
- ✅ 任务创建、分配、跟踪、干预、取消
- ✅ 看板式任务状态展示
- ✅ 实时消息流监控
- ✅ 权限请求审批
- ✅ Agent 工作目录隔离

**明确不在范围内**：
- ❌ Claude Code 核心引擎的修改
- ❌ 新 Agent 类型定义（仅使用现有类型）
- ❌ 自定义工具开发（仅配置工具访问权限）
- ❌ 多租户 SaaS 部署（仅单团队部署）
- ❌ 付费订阅管理

### 1.2 产品核心概念

#### 1.2.1 Agent（工作者）

**定义**：Agent 是 Claude Code 会话的抽象封装，代表一个正在工作的"虚拟人"。每个 Agent 拥有独立的工作目录、工具权限、对话历史和执行状态。

**Agent 作为"人"的隐喻**：

| "人"的属性 | Agent 对应能力 |
|------------|----------------|
| 身份 | agentId、name、color（唯一标识和可视化标签） |
| 能力 | AgentCapabilities（文件读写、命令执行、网络访问等） |
| 状态 | AgentStatus（idle/working/paused/error/offline） |
| 工作 | currentTaskId（当前正在执行的任务） |
| 历史 | AgentStatistics（完成任务数、Token 消耗、平均耗时） |
| 环境 | workDir（工作目录）、permissionMode（权限模式） |

**Agent 类型分类**：

```
┌─────────────────────────────────────────────────────────────┐
│                     Agent 类型体系                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Built-in    │  │ Custom      │  │ Fork        │         │
│  │ Agents      │  │ Agents      │  │ Agent       │         │
│  └─────┬───────┘  └─────┬───────┘  └─────┬───────┘         │
│        │                │                │                  │
│  ┌─────┴───────┐  ┌─────┴───────┐        │                  │
│  │general-     │  │用户/项目     │  继承父上下文              │
│  │purpose      │  │定义的Agent   │  权限冒泡机制              │
│  │Explore      │  │             │  共享Prompt缓存            │
│  │Plan         │  │             │                          │
│  │Verification │  │             │                          │
│  │fork         │  │             │                          │
│  └─────────────┘  └─────────────┘                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**内置 Agent 类型详解**：

| Agent 类型 | 功能定位 | 工具权限 | 特点 |
|------------|----------|----------|------|
| `general-purpose` | 通用研究 | 全工具访问 | 复杂搜索、深度分析 |
| `Explore` | 快速探索 | 只读工具 | omitClaudeMd，轻量上下文，haiku 模型 |
| `Plan` | 规划设计 | 只读工具 | 生成实施计划，架构设计 |
| `Verification` | 验证检查 | 文件读取 + Bash | 验证代码变更、测试执行 |
| `fork` | 继承执行 | 继承父工具 | 权限冒泡，共享 Prompt 缓存 |

#### 1.2.2 Task（任务）

**定义**：Task 是需要 Agent 执行的工作单元，包含任务描述、状态、依赖关系、执行记录等信息。

**Task 生命周期**：

```
┌─────────────────────────────────────────────────────────────────┐
│                     Task 状态流转                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│     ┌─────────┐     ┌─────────┐     ┌───────────┐     ┌───────┐│
│     │ pending │ ──→ │ blocked │ ──→ │in_progress│ ──→ │completed│ │
│     │ (待处理)│     │(被阻塞) │     │ (进行中)  │     │(已完成)││
│     └─────────┘     └─────────┘     └───────────┘     └───────┘│
│         │               │               │                    │ │
│         │               │               ↓                    │ │
│         │               │         ┌───────────┐              │ │
│         │               │         │ reviewing │ ──→ ────────→│ │
│         │               │         │ (待审核)  │              │ │
│         │               │         └───────────┘              │ │
│         │               │               │                    │ │
│         ↓               ↓               ↓                    │ │
│     ┌─────────────────────────────────────────────────────┐  │ │
│     │                cancelled / deleted                   │  │ │
│     │                (已取消 / 已删除)                     │  │ │
│     └─────────────────────────────────────────────────────┘  │ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Task 核心属性**：

| 属性类别 | 属性名 | 说明 |
|----------|--------|------|
| **基础标识** | id | 任务唯一 ID（数字递增） |
| | subject | 任务标题（祈使语气） |
| | description | 详细工作说明 |
| **状态信息** | status | 当前状态 |
| | owner | 所有者 Agent ID |
| | blocks | 此任务阻塞的任务列表 |
| | blockedBy | 阻塞此任务的任务列表 |
| **执行信息** | startedAt | 开始执行时间 |
| | completedAt | 完成时间 |
| | executionLog | 执行日志记录 |
| | result | 执行结果摘要 |
| **扩展属性** | priority | 优先级（高/中/低） |
| | tags | 标签分类 |
| | estimatedTime | 预估时间 |
| | deadline | 截止时间 |

#### 1.2.3 调度平台

**定义**：调度平台是协调多个 Agent 与 Task 的管理系统，提供创建、分配、监控、干预等核心功能。

**平台核心职责**：

1. **Agent 生命周期管理**
   - 创建 Agent（选择类型、配置权限、初始化工作目录）
   - 启动 Agent（通过 Bridge API 创建远程会话）
   - 监控 Agent（心跳检测、状态同步、资源统计）
   - 终止 Agent（优雅关闭、资源清理、结果归档）

2. **Task 生命周期管理**
   - 创建 Task（手动创建、AI 提取、批量导入）
   - 分配 Task（手动分配、自动匹配、Agent 声明）
   - 跟踪 Task（状态监控、进度追踪、日志收集）
   - 干预 Task（权限审批、方向调整、强制取消）

3. **实时通信管理**
   - 消息推送（Agent 状态、Task 状态、执行进度）
   - 权限请求（PreToolUse Hook 拦截、用户审批）
   - 控制指令（发送指令、中断信号、取消命令）

### 1.3 与现有 Claude Code 的关系

#### 1.3.1 上层封装定位

调度平台定位为 Claude Code 的**上层封装**，不修改 Claude Code 核心引擎，通过 Bridge API 和 Hook 系统实现集成。

**架构关系图**：

```
┌─────────────────────────────────────────────────────────────────┐
│                     调度平台架构定位                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                 调度平台（上层封装）                      │   │
│  │  ┌───────────────┐  ┌───────────────┐  ┌───────────────┐│   │
│  │  │ 看板界面      │  │ Agent 管理   │  │ Task 管理     ││   │
│  │  │ (React Web)  │  │ Service      │  │ Service       ││   │
│  │  └───────────────┘  └───────────────┘  └───────────────┘│   │
│  └─────────────────────────────────────────────────────────┘   │
│                          ↓ API 集成 ↓                          │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                 Claude Code 核心                         │   │
│  │  ┌───────────────┐  ┌───────────────┐  ┌───────────────┐│   │
│  │  │ AgentTool    │  │ Task Tools    │  │ Bridge API    ││   │
│  │  │ Hook System  │  │ AppState      │  │ Transport     ││   │
│  │  └───────────────┘  └───────────────┘  └───────────────┘│   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**不修改核心的原因**：
- 保持与 Claude Code 版本更新的兼容性
- 降低维护成本和复杂度
- 充分利用现有成熟功能

#### 1.3.2 复用能力清单

调度平台复用 Claude Code 的以下核心能力：

| 能力类别 | Claude Code 组件 | 调度平台用途 |
|----------|------------------|--------------|
| **Agent 启动** | `AgentTool`、`runAgent`、`registerAsyncAgent` | 创建和管理子 Agent |
| **Agent 类型** | `builtInAgents`、`loadAgentsDir` | 提供 Agent 类型选择 |
| **工具过滤** | `filterToolsForAgent`、`assembleToolPool` | 配置 Agent 工具权限 |
| **工作隔离** | `createAgentWorktree`、Worktree 机制 | Agent 工作目录隔离 |
| **Task 创建** | `TaskCreateTool`、`createTask` | 创建任务记录 |
| **Task 更新** | `TaskUpdateTool`、`updateTask` | 状态更新、分配 |
| **Task 依赖** | `blockTask`、blocks/blockedBy | 依赖关系管理 |
| **Task 声明** | `claimTask`、`claimTaskWithBusyCheck` | Agent 声明任务 |
| **Task 停止** | `TaskStopTool`、`stopTask` | 取消正在执行的任务 |
| **远程会话** | `createCodeSession`、`fetchRemoteCredentials` | 创建 Bridge 连接 |
| **传输层** | `createV2ReplTransport`、SSE | 实时消息通信 |
| **JWT 管理** | `createTokenRefreshScheduler` | 认证凭证刷新 |
| **权限拦截** | `PreToolUse` Hook、`PermissionRequest` Hook | 权限审批推送 |
| **事件通知** | `TaskCreated` Hook、`TaskCompleted` Hook | 任务事件监听 |
| **状态管理** | `AppState` Store 模式 | 前端状态管理 |
| **邮箱通信** | `writeToMailbox`、`SendMessageTool` | Agent 间消息传递 |

#### 1.3.3 集成方式概述

调度平台通过以下方式与 Claude Code 集成：

**1. Bridge API 集成**
- 通过 `createCodeSession` 创建远程会话
- 通过 `fetchRemoteCredentials` 获取 Worker JWT
- 通过 `createV2ReplTransport` 建立 SSE 连接
- 通过 `writeMessage` 发送控制指令

**2. Hook 系统集成**
- 注册 `PreToolUse` Hook 拦截权限请求
- 注册 `TaskCreated` Hook 监听任务创建
- 注册 `TaskCompleted` Hook 监听任务完成
- 注册 `PermissionRequest` Hook 处理权限审批

**3. 工具函数复用**
- 直接调用 `createTask`、`updateTask`、`claimTask` 等函数
- 复用 `runAgent`、`registerAsyncAgent` 启动 Agent
- 复用 `createAgentWorktree` 创建隔离工作目录

**4. 状态同步**
- 监听 `~/.claude/tasks/` 目录文件变更
- 复用 AppState 的 `tasks` 字段结构
- 通过 WebSocket/SSE 推送状态更新

## 第二章：功能需求详述

### 2.1 Agent 管理功能

#### 2.1.1 Agent 创建与配置

**创建入口**：

用户可通过以下方式创建 Agent：

| 创建方式 | 操作流程 | 适用场景 |
|----------|----------|----------|
| **连接现有会话** | 选择已运行的 Claude Code 会话 → 建立 Bridge 连接 | 复用已有工作上下文 |
| **新建独立 Agent** | 选择类型 → 配置工作目录 → 设置权限 → 启动 | 创建全新 Agent |
| **Fork 子 Agent** | 从父 Agent Fork → 继承上下文 → 独立执行 | 继承已有上下文的子任务 |

**Agent 类型选择界面**：

创建 Agent 时，用户可选择以下类型：

```
┌─────────────────────────────────────────────────────────────┐
│                     创建 Agent                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Agent 名称: [________________________]                     │
│                                                             │
│  Agent 类型:                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ ○ general-purpose  - 通用研究，全工具访问            │   │
│  │ ○ Explore          - 快速探索，只读模式，轻量上下文   │   │
│  │ ○ Plan             - 规划设计，只读模式，生成计划     │   │
│  │ ○ Verification     - 验证检查，验证代码变更          │   │
│  │ ○ fork             - Fork子Agent，继承父上下文       │   │
│  │ ○ Custom           - 用户自定义Agent                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  工作目录: [________________________] [浏览...]             │
│                                                             │
│  隔离模式: ○ 无隔离  ○ Worktree隔离                          │
│                                                             │
│  [取消]                              [创建并启动]           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**Agent 配置项详解**：

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| **name** | Agent 显示名称 | 用户自定义字符串 |
| **type** | Agent 类型 | general-purpose、Explore、Plan、Verification、fork、custom |
| **workDir** | 工作目录 | 本地路径，支持 Worktree 隔离 |
| **permissionMode** | 权限模式 | default、plan、auto、bypass、bubble（仅 fork） |
| **toolWhitelist** | 工具白名单 | 工具名称列表（如：Read、Grep、Bash） |
| **toolBlacklist** | 工具黑名单 | 工具名称列表（如：Edit、Write） |
| **mcpServers** | MCP 服务器配置 | MCP 配置对象列表 |
| **customSystemPrompt** | 自定义系统提示 | 字符串（仅 Custom Agent） |
| **isolation** | 隔离模式 | none、worktree |

**Worktree 隔离说明**：

当选择 `worktree` 隔离模式时：

```
主工作目录: /Users/dev/projects/myapp
Worktree目录: /Users/dev/projects/myapp/.claude/worktrees/agent-xxx

隔离效果:
- Agent 在独立 Git 工作树中执行
- 不影响主工作目录的文件状态
- 有变更时保留 Worktree，无变更自动清理
- 可在 Agent 完成后合并变更
```

```
│                    └──────────┘                                  │
│                                                                 │
│                    ┌──────────┐                                  │
│      Bridge断开───→│ offline  │                                  │
│                    └──────────┘                                  │
│                         │                                        │
│                     重连 │                                        │
│                         ↓                                        │
│                    ┌──────────┐                                  │
│                    │   idle   │                                  │
│                    └──────────┘                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**状态监控机制**：

| 监控项 | 监控方式 | 更新频率 | 处理方式 |
|--------|----------|----------|----------|
| **心跳检测** | Bridge 心跳（`heartbeatWork`） | 20 秒 | 超时标记 offline |
| **执行状态** | SSE 消息流推送 | 实时 | 解析消息更新状态 |
| **Token 消耗** | 累计 usage 字段 | 每条消息 | 实时统计展示 |
| **工具调用** | tool_use 消息解析 | 实时 | 记录调用次数 |
| **执行时长** | startedAt 时间戳 | 每秒更新 | 计算差值展示 |

#### 2.1.3 Agent 生命周期操作

**启动 Agent**：

```
启动流程:
1. 用户点击"启动"按钮
2. 调用 Bridge API createCodeSession 创建会话
3. 获取 Worker JWT (fetchRemoteCredentials)
4. 建立 SSE 传输连接 (createV2ReplTransport)
5. 启动 Agent 执行循环 (runAgent)
6. 更新 Agent 状态为 working
7. 开始接收消息流
```

**暂停 Agent**：

```
暂停流程:
1. 用户点击"暂停"按钮
2. 发送中断信号 (interrupt)
3. Agent 当前操作暂停
4. 保留当前对话上下文
5. 状态更新为 paused
6. 显示暂停原因和恢复选项
```

**恢复 Agent**：

```
恢复流程:
1. 用户点击"恢复"按钮或输入指令
2. 从 Transcript 恢复上下文
3. 发送用户消息继续执行
4. 状态更新为 working
5. 继续接收消息流
```

**终止 Agent**：

```
终止流程:
1. 用户点击"终止"按钮
2. 发送取消信号 (TaskStop)
3. Agent 优雅关闭（完成当前工具调用）
4. 归档会话 (archiveSession)
5. 清理资源（关闭 MCP 连接、清理 Worktree）
6. 状态更新为 completed
7. 显示执行结果摘要
```

#### 2.2.2 Task 状态流转

**标准状态定义**：

| 状态 | 描述 | 状态特征 |
|------|------|----------|
| `pending` | 待处理，等待分配 | 无 owner，未被阻塞 |
| `blocked` | 被阻塞，等待依赖完成 | blockedBy 任务未完成 |
| `in_progress` | 进行中，正在执行 | 有 owner，startedAt 已记录 |
| `reviewing` | 等待审核，执行完成待确认 | 执行完成，需用户审核 |
| `completed` | 已完成，审核通过 | completedAt 已记录，依赖解除 |
| `cancelled` | 已取消，用户主动取消 | 记录取消原因和时间 |
| `deleted` | 已删除，永久移除 | 从列表中不再显示 |

**状态约束规则**：

| 规则 | 说明 |
|------|------|
| **阻塞检查** | blockedBy 任务未完成时禁止进入 in_progress |
| **声明检查** | 已有 owner 时禁止其他 Agent 声明 |
| **完成验证** | 进入 completed 状态需执行 TaskCompleted Hook |
| **取消清理** | 取消时需清理依赖引用 |

#### 2.2.3 Task 依赖关系

**依赖类型**：

| 依赖方向 | 字段名 | 说明 |
|----------|--------|------|
| **正向阻塞** | `blocks` | 此任务阻塞的目标任务（此任务完成后才能开始目标任务） |
| **反向阻塞** | `blockedBy` | 阻塞此任务的来源任务（来源任务完成后才能开始此任务） |

**依赖建立操作**：

```
建立依赖:
1. 用户选择"被阻塞于"或"阻塞"选项
2. 输入目标任务 ID
3. 调用 blockTask(fromId, toId) 建立双向关系
4. 更新两任务的 blocks/blockedBy 字段
5. 检测循环依赖并警告

移除依赖:
1. 用户点击依赖关系删除按钮
2. 更新两任务的 blocks/blockedBy 字段
3. 检查是否解除阻塞状态
```

#### 2.2.4 Task 分配机制

**手动分配**：

用户直接指定 Task 的 owner：

```
分配流程:
1. 用户点击任务卡片
2. 选择"分配给 Agent"
3. 从 Agent 列表选择目标 Agent
4. 检查 Agent 是否空闲
5. 更新 Task owner 字段
6. 通过 writeToMailbox 通知 Agent
```

**自动分配**：

系统自动匹配最合适的 Agent：

| 分配策略 | 匹配规则 | 适用场景 |
|----------|----------|----------|
| **能力匹配** | 根据 AgentCapabilities 与 requiredCapabilities 匹配 | 需要特定能力的任务 |
| **负载均衡** | 选择当前任务数最少的 Agent | 大量任务并行执行 |
| **依赖就近** | 优先分配给已完成依赖任务的 Agent | 依赖关系密集的任务 |
| **历史表现** | 根据 AgentStatistics 评分选择 | 追求效率的任务 |

**Agent 声明机制**：

Agent 可主动声明任务（claimTask）：

```
声明流程:
1. Agent 检查任务状态（pending，无 owner）
2. 检查阻塞依赖是否解除
3. 检查自身是否忙碌（无其他 in_progress 任务）
4. 调用 claimTask 声明所有权
5. 更新 Task owner 字段
6. 开始执行任务
```

**声明失败处理**：

| 失败原因 | 处理方式 |
|----------|----------|
| `task_not_found` | 显示错误，任务不存在 |
| `already_claimed` | 显示警告，任务已被分配 |
| `already_resolved` | 显示通知，任务已完成 |
| `blocked` | 显示阻塞任务列表，等待依赖 |
| `agent_busy` | 显示忙碌任务列表，等待空闲 |

PlatformAgent 是调度平台中 Agent 的完整数据模型，扩展了 Claude Code 的 AgentDefinition。

#### 3.1.1 基础结构定义

```typescript
/**
 * PlatformAgent - 调度平台 Agent 实体定义
 * 扩展 Claude Code AgentDefinition，增加状态管理和统计信息
 */
interface PlatformAgent {
  // ==================== 基础标识 ====================
  
  /** Agent 唯一 ID，格式: agent-{timestamp}-{random} */
  id: string
  
  /** 用户可见显示名称 */
  name: string
  
  /** 颜色标识（用于 UI 显示），格式: #RRGGBB */
```

```typescript
interface AgentStatistics {

  /** 最长完成时间（秒） */
  maxCompletionTime: number
  
  /** 总执行时间（秒） */

  /** 总 Token 消耗 */

  /** 输入 Token 消耗 */

  /** 输出 Token 消耗 */

  /** 平均每任务 Token 消耗 */
  avgTokensPerTask: number
  
  /** 平均每任务工具调用次数 */

}

/**
 * 计算效率指标
 */
function calculateStatistics(stats: Partial<AgentStatistics>): AgentStatistics {
  const totalTasks = stats.tasksCompleted + stats.tasksFailed + stats.tasksCancelled
  const successRate = totalTasks > 0 
    ? (stats.tasksCompleted / totalTasks) * 100 
    : 0
  
  const avgTokensPerTask = stats.tasksCompleted > 0 
    ? stats.totalTokensUsed / stats.tasksCompleted 
    : 0
  
  const avgToolCallsPerTask = stats.tasksCompleted > 0 
    ? stats.totalToolCalls / stats.tasksCompleted 
    : 0
  
  return {
    tasksCompleted: stats.tasksCompleted ?? 0,
    tasksFailed: stats.tasksFailed ?? 0,
    tasksCancelled: stats.tasksCancelled ?? 0,
    tasksInProgress: stats.tasksInProgress ?? 0,
    avgCompletionTime: stats.avgCompletionTime ?? 0,
    minCompletionTime: stats.minCompletionTime ?? 0,
    maxCompletionTime: stats.maxCompletionTime ?? 0,
    totalExecutionTime: stats.totalExecutionTime ?? 0,
    totalTokensUsed: stats.totalTokensUsed ?? 0,
    inputTokensUsed: stats.inputTokensUsed ?? 0,
    outputTokensUsed: stats.outputTokensUsed ?? 0,
    totalToolCalls: stats.totalToolCalls ?? 0,
    lastActiveAt: stats.lastActiveAt ?? new Date(),
    totalActiveDuration: stats.totalActiveDuration ?? 0,
    successRate,
    avgTokensPerTask,
    avgToolCallsPerTask
  }
}
```

### 5.4 交互设计

#### 5.4.1 拖拽分配

**完整交互流程**：

```
1. 用户点击任务卡片
2. 卡片显示拖拽手柄
3. 用户拖拽卡片
4. 系统显示拖拽预览

   - 侧边栏 Agent 高亮显示
5. 用户悬停在目标 Agent
   - Agent 卡片边框变绿
   - 显示提示: "将分配给 Agent-Explore"
6. 用户释放鼠标
   - 调用 TaskService.assignTask
   - 更新任务 owner
   - 通过 writeToMailbox 通知 Agent
   - 显示成功提示: "已分配给 Agent-Explore"
```

**拖拽验证**：

| 验证项 | 失败提示 |
|--------|----------|
| Agent 状态非 idle/paused | "Agent 正在执行其他任务" |
| Agent 能力不匹配 | "Agent 无所需能力" |
| 任务已被分配 | "任务已分配给其他 Agent" |
| 任务状态非 pending | "任务不在可分配状态" |

#### 5.4.2 实时更新机制

**WebSocket/SSE 推送**：

```typescript
// 实时消息类型
interface RealtimeMessage {
  type: RealtimeMessageType
  payload: unknown
  timestamp: Date
}

type RealtimeMessageType =
  | 'agent_status_change'    // Agent 状态变更
  | 'task_status_change'     // Task 状态变更
  | 'task_progress_update'   // Task 进度更新
  | 'agent_message'          // Agent 执行消息
  | 'permission_request'     // 权限审批请求
  | 'task_completed'         // 任务完成通知
  | 'connection_status'      // 连接状态变更
```

**状态更新处理**：

| 消息类型 | 前端处理 | UI 更新 |
|----------|----------|----------|
| `agent_status_change` | 更新 store.agents[id].status | 刷新侧边栏卡片 |
| `task_status_change` | 更新 store.tasks[id].status | 刷新看板卡片位置 |
| `task_progress_update` | 更新 store.tasks[id].progress | 刷新进度条 |
| `agent_message` | append to executionLog | 刷新详情面板日志 |
| `permission_request` | 显示审批弹窗 | 弹窗请求用户决策 |
| `task_completed` | 显示完成通知 | 弹窗显示结果摘要 |

**断线重连机制**：

```
连接断开时:
1. 显示连接状态为 "断开"
2. 尝试自动重连（最多 5 次）
3. 重连间隔: 1s, 2s, 4s, 8s, 16s（指数退避）
4. 重连成功后:
   - 重新同步状态
   - 显示连接恢复通知
5. 重连失败后:
   - 显示 "连接已断开，请刷新页面"
   - 提供 "手动刷新" 按钮
```

#### 6.2.1 Agent 创建集成

**集成 AgentTool 启动子 Agent**：

```typescript
// backend/src/integration/agentToolIntegration.ts
import { runAgent, registerAsyncAgent } from 'claude-code/tools/AgentTool'
import { createCodeSession, fetchRemoteCredentials } from 'claude-code/bridge'
import { createV2ReplTransport } from 'claude-code/bridge/replBridgeTransport'
import { builtInAgents } from 'claude-code/tools/AgentTool/builtInAgents'
import { createAgentWorktree } from 'claude-code/utils/worktree'
import { filterToolsForAgent, assembleToolPool } from 'claude-code/tools/AgentTool/agentToolUtils'

/**
 * Agent 创建集成服务
 */
export class AgentToolIntegration {
  
  /**
   * 创建 PlatformAgent
   * @param request 创建请求
   * @param userId 用户 ID
   * @returns Agent 创建结果
   */
  async createAgent(
    request: CreateAgentRequest,
    userId: string
  ): Promise<AgentResponse> {
    
    // Step 1: 获取 Agent 定义
    const selectedAgent = this.getAgentDefinition(request.type, request.customSystemPrompt)
    
    // Step 2: 创建远程会话
    const sessionId = await createCodeSession(
      this.baseUrl,
      this.accessToken,
      request.name,
      this.timeout,
      ['platform-agent', request.type]
    )
    
    // Step 3: 获取 Bridge 凭证
    const credentials = await fetchRemoteCredentials(
      sessionId,
      this.baseUrl,
      this.accessToken,
      this.timeout
    )
    
    // Step 4: 建立 SSE 传输连接
    const transport = await createV2ReplTransport({
      sessionUrl: buildCCRv2SdkUrl(credentials.api_base_url, sessionId),
      ingressToken: credentials.worker_jwt,
      sessionId,
      epoch: credentials.worker_epoch,
      heartbeatIntervalMs: 20000,
      outboundOnly: false
    })
    
    // Step 5: 创建 Worktree（如需要）
    let worktreeInfo: WorktreeInfo | null = null
    if (request.isolation === 'worktree') {
      const slug = `agent-${Date.now().toString(36)}`
      worktreeInfo = await createAgentWorktree(slug)
    }
    
    // Step 6: 组装工具池
    const workerPermissionContext = {
      mode: request.permissionMode || 'default',
      // ... 其他权限配置
    }
    const workerTools = assembleToolPool(workerPermissionContext, this.mcpTools)
    const filteredTools = filterToolsForAgent({
      tools: workerTools,
      isBuiltIn: request.type !== 'custom',
      isAsync: true,
      permissionMode: workerPermissionContext.mode
    })
    
    // Step 7: 创建 Agent ID
    const agentId = createAgentId()
    
    // Step 8: 注册异步 Agent
    const taskState = registerAsyncAgent({
      agentId,
      description: request.name,
      prompt: request.initialPrompt || '',
      selectedAgent,
      setAppState: this.rootSetAppState,
      toolUseId: generateToolUseId()
    })
    
    // Step 9: 存储到 Agent Registry
    const platformAgent: PlatformAgent = {
      id: agentId,
      name: request.name,
      color: assignAgentColor(agentId),
      type: request.type,
      capabilities: this.buildCapabilities(selectedAgent),
      workDir: request.workDir || worktreeInfo?.worktreePath || process.cwd(),
      permissionMode: request.permissionMode || 'default',
      toolWhitelist: request.toolWhitelist || [],
      toolBlacklist: request.toolBlacklist || [],
      mcpServers: request.mcpServers || [],
      status: 'idle',
      currentTaskId: undefined,
      load: 0,
      statistics: {
        tasksCompleted: 0,
        tasksFailed: 0,
        avgCompletionTime: 0,
        totalTokensUsed: 0,
        totalToolCalls: 0,
        lastActiveAt: new Date()
      },
      sessionId,
      bridgeUrl: credentials.api_base_url,
      workerJwt: credentials.worker_jwt,
      jwtExpiresAt: new Date(Date.now() + credentials.expires_in * 1000),
      workerEpoch: credentials.worker_epoch,
      createdAt: new Date(),
      createdBy: userId,
      updatedAt: new Date()
    }
    
    await this.agentRegistry.save(platformAgent)
    
    // Step 10: 启动传输连接监听
    transport.setOnData((data) => {
      this.handleInboundMessage(sessionId, data)
    })
    transport.connect()
    this.transports.set(sessionId, transport)
    
    return {
      agent: platformAgent,
      bridgeUrl: buildWebUrl(sessionId),
      connectionStatus: 'connected'
    }
  }
  
  /**
   * 获取 Agent 定义
   */
  private getAgentDefinition(
    type: AgentType,
    customPrompt?: string
  ): AgentDefinition {
    if (type === 'custom' && customPrompt) {
      return {
        agentType: 'custom',
        whenToUse: 'User defined custom agent',
        source: 'custom',
        getSystemPrompt: () => customPrompt,
        model: 'inherit',
        maxTurns: 200
      }
    }
    
    const builtIn = builtInAgents.find(a => a.agentType === type)
    if (!builtIn) {
      throw new Error(`Unknown agent type: ${type}`)
    }
    return builtIn
  }
  
  /**
   * 构建能力描述
   */
  private buildCapabilities(agentDef: AgentDefinition): AgentCapabilities {
    const disallowed = agentDef.disallowedTools || []
    return {
      canRead: !disallowed.includes('Read'),
      canWrite: !disallowed.includes('Edit') && !disallowed.includes('Write'),
      canExecute: !disallowed.includes('Bash'),
      canSearch: !disallowed.includes('Grep') && !disallowed.includes('Glob'),
      canWeb: !disallowed.includes('WebFetch') && !disallowed.includes('WebSearch'),
      canMCP: [],
      maxTurns: agentDef.maxTurns || 200,
      canSpawnSubagent: !disallowed.includes('AgentTool'),
      canStopOthers: false
    }
  }
}
```

#### 6.2.2 Task 创建集成

**集成 TaskCreateTool 创建任务**：

```typescript
// backend/src/integration/taskToolIntegration.ts
import { createTask, updateTask, blockTask, claimTask, deleteTask } from 'claude-code/utils/tasks'
import { executeTaskCreatedHooks, executeTaskCompletedHooks } from 'claude-code/utils/hooks'
import { writeToMailbox } from 'claude-code/utils/teammateMailbox'

/**
 * Task 工具集成服务
 */
export class TaskToolIntegration {
  
  private taskListId: string
  
  constructor(taskListId: string) {
    this.taskListId = taskListId
  }
  
  /**
   * 创建任务（扩展 TaskCreateTool）
   */
  async createTask(request: CreateTaskRequest): Promise<TaskResponse> {
    
    // Step 1: 调用 createTask 创建基础任务
    const taskId = await createTask(this.taskListId, {
      subject: request.subject,
      description: request.description,
      activeForm: request.activeForm,
      status: 'pending',
      owner: undefined,
      blocks: [],
      blockedBy: request.blockedBy || [],
      metadata: {
        priority: request.priority || 'medium',
        tags: request.tags || [],
        estimatedTime: request.estimatedTime,
        deadline: request.deadline,
        preferredAgent: request.preferredAgent,
        requiredCapabilities: request.requiredCapabilities,
        createdBy: request.createdBy,
        createdAt: new Date().toISOString()
      }
    })
    
    // Step 2: 建立依赖关系
    if (request.blockedBy?.length) {
      for (const blockerId of request.blockedBy) {
        await blockTask(this.taskListId, blockerId, taskId)
      }
    }
    
    if (request.blocks?.length) {
      for (const blockedId of request.blocks) {
        await blockTask(this.taskListId, taskId, blockedId)
      }
    }
    
    // Step 3: 执行 TaskCreated Hooks（用于验证）
    const blockingErrors: string[] = []
    const hookGenerator = executeTaskCreatedHooks(
      taskId,
      request.subject,
      request.description,
      getAgentName(),
      getTeamName(),
      undefined,
      undefined,
      undefined,
      this.context
    )
    
    for await (const result of hookGenerator) {
      if (result.blockingError) {
        blockingErrors.push(result.blockingError)
      }
    }
    
    if (blockingErrors.length > 0) {
      // Hook 拒绝创建，删除任务
      await deleteTask(this.taskListId, taskId)
      return {
        success: false,
        taskId,
        error: blockingErrors.join('\n'),
        operationAt: new Date()
      }
    }
    
    // Step 4: 通知广播
    this.broadcastTaskCreated(taskId)
    
    return {
      success: true,
      taskId,
      task: await this.getPlatformTask(taskId),
      operationAt: new Date()
    }
  }
  
  /**
   * 分配任务给 Agent
   */
  async assignTask(taskId: string, agentId: string): Promise<TaskResponse> {
    
    // Step 1: 更新 owner
    const updated = await updateTask(this.taskListId, taskId, {
      owner: agentId
    })
    
    if (!updated) {
      return {
        success: false,
        taskId,
        error: 'Task not found',
        operationAt: new Date()
      }
    }
    
    // Step 2: 发送邮箱通知（复用 writeToMailbox）
    const task = await this.getPlatformTask(taskId)
    const assignmentMessage = JSON.stringify({
      type: 'task_assignment',
      taskId,
      subject: task.subject,
      description: task.description,
      assignedBy: 'platform',
      timestamp: new Date().toISOString()
    })
    
    await writeToMailbox(
      agentId,
      {
        from: 'platform',
        text: assignmentMessage,
        timestamp: new Date().toISOString()
      },
      this.taskListId
    )
    
    // Step 3: 更新 Agent 负载
    await this.updateAgentLoad(agentId, 1)
    
    return {
      success: true,
      taskId,
      task: await this.getPlatformTask(taskId),
      updatedFields: ['owner'],
      operationAt: new Date()
    }
  }
  
  /**
   * Agent 声明任务（复用 claimTask）
   */
  async claimTask(taskId: string, agentId: string): Promise<ClaimResult> {
    
    const result = await claimTask(this.taskListId, taskId, agentId)
    
    if (result.success) {
      // 更新 Agent 负载
      await this.updateAgentLoad(agentId, 1)
      // 广播状态变更
      this.broadcastTaskClaimed(taskId, agentId)
    }
    
    return result
  }
  
  /**
   * 取消任务（复用 TaskStop + deleteTask）
   */
  async cancelTask(taskId: string, reason?: string): Promise<TaskResponse> {
    
    const task = await this.getPlatformTask(taskId)
    
    // Step 1: 如果有正在执行的 Agent，先停止
    if (task.owner && task.status === 'in_progress') {
      await this.stopAgentTask(task.owner, taskId)
    }
    
    // Step 2: 更新状态为 cancelled（使用 updateTask metadata）
    await updateTask(this.taskListId, taskId, {
      metadata: {
        cancelReason: reason,
        cancelledAt: new Date().toISOString(),
        cancelledBy: 'platform'
      }
    })
    
    // Step 3: 删除任务（标记为 deleted）
    await deleteTask(this.taskListId, taskId)
    
    // Step 4: 清理依赖引用
    // deleteTask 已自动清理
    
    return {
      success: true,
      taskId,
      statusChange: { from: task.status, to: 'cancelled' },
      operationAt: new Date()
    }
  }
}
```

```typescript
// backend/src/hooks/platformHooks.ts
import { registerHook, executePreToolHooks } from 'claude-code/utils/hooks'

/**
 * 注册平台自定义 Hooks
 */
export function registerPlatformHooks(context: PlatformContext): void {
  
  // === TaskCreated Hook: 监控任务创建 ===
  registerHook({
    type: 'callback',
    event: 'TaskCreated',
    callback: async (input) => {
      // 推送到前端 WebSocket
      context.broadcast({
        type: 'task_created',
        task: {
          id: input.taskId,
          subject: input.subject,
          description: input.description
        }
      })
      return { succeeded: true }
    }
  })
  
  // === TaskCompleted Hook: 监控任务完成 ===
  registerHook({
    type: 'callback',
    event: 'TaskCompleted',
    callback: async (input) => {
      // 更新 Agent 统计
      await context.updateAgentStatistics(input.agentName, {
        tasksCompleted: 1,
        lastActiveAt: new Date()
      })
      
      // 推送到前端
      context.broadcast({
        type: 'task_completed',
        taskId: input.taskId,
        summary: input.summary
      })
      
      return { succeeded: true }
    }
  })
  
  // === PreToolUse Hook: 权限审批推送 ===
  registerHook({
    type: 'callback',
    event: 'PreToolUse',
    callback: async (input) => {
      // 检查是否需要用户审批
      if (shouldRequestUserApproval(input.tool_name, input.tool_input)) {
        // 推送权限请求到前端
        const requestId = generateRequestId()
        context.broadcast({
          type: 'permission_request',
          requestId,
          agentId: input.agentId,
          toolName: input.tool_name,
          toolInput: input.tool_input
        })
        
        // 等待用户决策
        const decision = await context.waitForPermissionDecision(requestId)
        
        return {
          succeeded: true,
          permissionDecision: decision.decision, // 'allow' | 'deny' | 'ask'
          updatedInput: decision.modifiedInput
        }
      }
      
      return { succeeded: true, permissionDecision: 'allow' }
    }
  })
  
  // === PermissionRequest Hook: 处理权限请求 ===
  registerHook({
    type: 'callback',
    event: 'PermissionRequest',
    callback: async (input) => {
      // 推送权限请求到前端
      const requestId = generateRequestId()
      context.broadcast({
        type: 'permission_request',
        requestId,
        agentId: input.agentId,
        permissionType: input.permission_type,
        message: input.message
      })
      
      // 等待用户决策
      const decision = await context.waitForPermissionDecision(requestId)
      
      return {
        succeeded: true,
        permissionDecision: decision.decision
      }
    }
  })
  
  // === SubagentStart Hook: Agent 启动通知 ===
  registerHook({
    type: 'callback',
    event: 'SubagentStart',
    callback: async (input) => {
      context.broadcast({
        type: 'agent_started',
        agentId: input.agentId,
        agentType: input.agentType,
        description: input.description
      })
      return { succeeded: true }
    }
  })
  
  // === SubagentStop Hook: Agent 结束通知 ===
  registerHook({
    type: 'callback',
    event: 'SubagentStop',
    callback: async (input) => {
      // 更新 Agent 状态
      await context.updateAgentStatus(input.agentId, 'idle')
      
      context.broadcast({
        type: 'agent_stopped',
        agentId: input.agentId,
        status: input.status,
        summary: input.summary
      })
      return { succeeded: true }
    }
  })
}

/**
 * 判断是否需要用户审批
 */
function shouldRequestUserApproval(toolName: string, toolInput: unknown): boolean {
  const sensitiveTools = ['Edit', 'Write', 'Bash', 'NotebookEdit']
  const destructivePatterns = [
    { tool: 'Bash', patterns: ['rm', 'delete', 'DROP', 'truncate'] },
    { tool: 'Write', patterns: [] } // 所有写入都需要审批
  ]
  
  if (sensitiveTools.includes(toolName)) {
    return true
  }
  
  // 检查破坏性命令
  if (toolName === 'Bash' && typeof toolInput === 'object') {
    const command = (toolInput as any).command || ''
    for (const pattern of destructivePatterns.find(d => d.tool === 'Bash')?.patterns || []) {
      if (command.includes(pattern)) {
        return true
      }
    }
  }
  
  return false
```

```typescript
// frontend/src/services/realtimeClient.ts

/**
 * RealtimeClient - 实时通信客户端
 */
export class RealtimeClient {
  private ws: WebSocket | null = null
  private reconnectAttempts = 0
  private maxReconnectAttempts = 5
  private reconnectDelay = 1000
  private messageQueue: string[] = []
  private subscriptions: Map<string, (data: unknown) => void> = new Map()
  
  constructor(private baseUrl: string) {}
  
  /**
   * 建立 WebSocket 连接
   */
  connect(): Promise<void> {
    return new Promise((resolve, reject) => {
      const wsUrl = `${this.baseUrl}/realtime`
      this.ws = new WebSocket(wsUrl)
      
      this.ws.onopen = () => {
        console.log('WebSocket connected')
        usePlatformStore.getState().setConnectionStatus('connected')
        this.reconnectAttempts = 0
        
        // 发送队列中的消息
        while (this.messageQueue.length > 0) {
          const msg = this.messageQueue.shift()
          if (msg) this.ws?.send(msg)
        }
        
        resolve()
      }
      
      this.ws.onmessage = (event) => {
        try {
          const data = JSON.parse(event.data)
          this.handleMessage(data)
        } catch (error) {
          console.error('Failed to parse message:', error)
        }
      }
      
      this.ws.onclose = (event) => {
        console.log('WebSocket closed:', event.code, event.reason)
        usePlatformStore.getState().setConnectionStatus('disconnected')
        
        // 尝试重连
        if (this.reconnectAttempts < this.maxReconnectAttempts) {
          this.scheduleReconnect()
        }
      }
      
      this.ws.onerror = (error) => {
        console.error('WebSocket error:', error)
        reject(error)
      }
    })
  }
  
  /**
   * 处理接收的消息
   */
  private handleMessage(data: RealtimeMessage): void {
    const store = usePlatformStore.getState()
    
    switch (data.type) {
      case 'agent_status_change':
        store.updateAgent(data.payload.agentId, {
          status: data.payload.status,
          currentTaskId: data.payload.currentTaskId
        })
        break
      
      case 'task_status_change':
        store.updateTask(data.payload.taskId, {
          status: data.payload.status,
          owner: data.payload.owner
        })
        break
      
      case 'task_progress_update':
        store.updateTask(data.payload.taskId, {
          // 更新进度相关字段
        })
        break
      
      case 'agent_message':
        // 添加到执行日志
        this.appendToExecutionLog(data.payload.agentId, data.payload.message)
        break
      
      case 'permission_request':
        store.addPermissionRequest({
          requestId: data.payload.requestId,
          agentId: data.payload.agentId,
          toolName: data.payload.toolName,
          toolInput: data.payload.toolInput,
          timestamp: new Date()
        })
        break
      
      case 'task_completed':
        // 显示完成通知
        this.showTaskCompletedNotification(data.payload)
        break
      
      case 'connection_status':
        store.setConnectionStatus(data.payload.status)
        break
      
      default:
        // 调用订阅的回调
        const callback = this.subscriptions.get(data.type)
        if (callback) callback(data.payload)
    }
    
    store.setState({ lastMessageAt: new Date() })
  }
  
  /**
   * 发送消息
   */
  send(type: string, payload: unknown): void {
    const message = JSON.stringify({ type, payload, timestamp: Date.now() })
    
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(message)
    } else {
      // 队列等待连接恢复
      this.messageQueue.push(message)
    }
  }
  
  /**
   * 发送权限决策
   */
  sendPermissionDecision(requestId: string, decision: PermissionDecision): void {
    this.send('permission_decision', {
      requestId,
      decision: decision.decision,
      comment: decision.comment,
      modifiedInput: decision.modifiedInput
    })
  }
  
  /**
   * 计划重连
   */
  private scheduleReconnect(): void {
    this.reconnectAttempts++
    const delay = this.reconnectDelay * Math.pow(2, this.reconnectAttempts - 1)
    
    usePlatformStore.getState().setConnectionStatus('reconnecting')
    
    setTimeout(() => {
      console.log(`Reconnecting... attempt ${this.reconnectAttempts}`)
      this.connect().catch(console.error)
    }, delay)
  }
  
  /**
   * 关闭连接
   */
  disconnect(): void {
    if (this.ws) {
      this.ws.close()
      this.ws = null
    }
  }
  
  /**
   * 订阅特定消息类型
   */
  subscribe(type: string, callback: (data: unknown) => void): () => void {
    this.subscriptions.set(type, callback)
    return () => this.subscriptions.delete(type)
  }
}
```

```typescript
// backend/src/store/taskStoreSync.ts
import fs from 'fs'
import path from 'path'
import { getTasksDir, listTasks, getTask } from 'claude-code/utils/tasks'

/**
 * Task Store 文件监听同步服务
 */
export class TaskStoreSync {
  private watchers: Map<string, fs.FSWatcher> = new Map()
  
  /**
   * 开始监听任务目录
   */
  startWatching(taskListId: string): void {
    const tasksDir = getTasksDir(taskListId)
    
    const watcher = fs.watch(tasksDir, (eventType, filename) => {
      if (eventType === 'change' && filename?.endsWith('.json')) {
        const taskId = filename.replace('.json', '')
        this.syncTaskToState(taskListId, taskId)
      }
      
      if (eventType === 'rename' && filename?.endsWith('.json')) {
        // 文件创建或删除
        this.syncTaskList(taskListId)
      }
    })
    
    this.watchers.set(taskListId, watcher)
  }
  
  /**
   * 同步单个任务到状态
   */
  private async syncTaskToState(taskListId: string, taskId: string): void {
    const task = await getTask(taskListId, taskId)
    
    if (task) {
      // 转换为 PlatformTask
      const platformTask = this.toPlatformTask(task)
      
      // 更新前端状态
      usePlatformStore.getState().updateTask(taskId, platformTask)
      
      // 广播变更
      this.broadcast({
        type: 'task_updated',
        taskId,
        task: platformTask
      })
    }
  }
  
  /**
   * 同步整个任务列表
   */
  private async syncTaskList(taskListId: string): void {
    const tasks = await listTasks(taskListId)
    
    const taskMap: Record<string, PlatformTask> = {}
    for (const task of tasks) {
      taskMap[task.id] = this.toPlatformTask(task)
    }
    
    usePlatformStore.getState().setTasks(taskMap)
    
    this.broadcast({
      type: 'tasks_synced',
      tasks: taskMap
    })
  }
  
  /**
   * 转换为 PlatformTask
   */
  private toPlatformTask(task: Task): PlatformTask {
    return {
      id: task.id,
      subject: task.subject,
      description: task.description,
      activeForm: task.activeForm,
      priority: (task.metadata?.priority as TaskPriority) || 'medium',
      tags: (task.metadata?.tags as string[]) || [],
      estimatedTime: task.metadata?.estimatedTime as number,
      deadline: task.metadata?.deadline as Date,
      status: task.status as ExtendedTaskStatus,
      owner: task.owner,
      blocks: task.blocks,
      blockedBy: task.blockedBy,
      executionLog: [],
      metadata: task.metadata || {},
      createdAt: new Date(task.metadata?.createdAt as string || Date.now()),
      createdBy: task.metadata?.createdBy as string || 'unknown',
      updatedAt: new Date()
    }
  }
  
  /**
   * 停止监听
   */
  stopWatching(taskListId: string): void {
    const watcher = this.watchers.get(taskListId)
    if (watcher) {
      watcher.close()
      this.watchers.delete(taskListId)
    }
  }
}
```

```typescript
import { createTokenRefreshScheduler } from 'claude-code/bridge/jwtUtils'

/**
 * Agent 状态同步服务
 */
export class AgentStatusSync {
  private tokenSchedulers: Map<string, TokenRefreshScheduler> = new Map()
  private heartbeatTimers: Map<string, NodeJS.Timeout> = new Map()
  
  /**
   * 注册 Agent 心跳检测
   */
  registerAgentHeartbeat(agentId: string, sessionId: string): void {
    // 心跳检测定时器
    const heartbeatTimer = setInterval(() => {
      this.checkAgentHeartbeat(agentId, sessionId)
    }, 20000) // 20 秒检测一次
    
    this.heartbeatTimers.set(agentId, heartbeatTimer)
  }
  
  /**
   * 检查 Agent 心跳
   */
  private async checkAgentHeartbeat(agentId: string, sessionId: string): void {
    const transport = this.transports.get(sessionId)
    
    if (!transport || transport.status !== 'connected') {
      // 连接断开，更新状态
      usePlatformStore.getState().updateAgent(agentId, {
        status: 'offline'
      })
      
      this.broadcast({
        type: 'agent_status_change',
        agentId,
        status: 'offline'
      })
    }
  }
  
  /**
   * 注册 JWT 刷新调度器
   */
  registerJwtRefresh(agentId: string, credentials: RemoteCredentials): void {
    const scheduler = createTokenRefreshScheduler({
      sessionId: credentials.sessionId,
      baseUrl: credentials.api_base_url,
      accessToken: this.accessToken,
      expires_in: credentials.expires_in,
      worker_epoch: credentials.worker_epoch,
      onRefresh: (newCredentials) => {
        // 更新存储的凭证
        usePlatformStore.getState().updateAgent(agentId, {
          workerJwt: newCredentials.worker_jwt,
          jwtExpiresAt: new Date(Date.now() + newCredentials.expires_in * 1000),
          workerEpoch: newCredentials.worker_epoch
        })
      },
      onError: (error) => {
        console.error(`JWT refresh failed for agent ${agentId}:`, error)
        // 标记为需要重连
        usePlatformStore.getState().updateAgent(agentId, {
          status: 'offline'
        })
      }
    })
    
    this.tokenSchedulers.set(agentId, scheduler)
    scheduler.start()
  }
  
  /**
   * 清理 Agent 同步
   */
  cleanupAgent(agentId: string): void {
    // 停止心跳检测
    const heartbeatTimer = this.heartbeatTimers.get(agentId)
    if (heartbeatTimer) {
      clearInterval(heartbeatTimer)
      this.heartbeatTimers.delete(agentId)
    }
    
    // 停止 JWT 刷新
    const scheduler = this.tokenSchedulers.get(agentId)
    if (scheduler) {
      scheduler.stop()
      this.tokenSchedulers.delete(agentId)
    }
  }
}
```
